# Выкуп по Клиентам

## Цель
Разработка отчёта для оценки уровня выкупа по каждому клиенту в разрезе регионов.

## Основные задачи
- Создать новый раздел в Staff: **Отчёты - Выкуп по Клиентам**

- Добавить фильтрацию по периоду (по дате вручения), клиенту и базе (Мск / Спб).

- Предусмотреть сортировку по всем столбцам таблицы (А-Я / Я-А)

- Добавить кнопку **"Скачать"** с выгрузкой общего отчёта и отчёта по регионам

- Добавить кнопку **"Контроль доставляемости"** для настройки пороговых значений и отображения предупреждений

## Верхняя часть страницы
В верхней части страницы расположены фильтры и кнопки **"Скачать отчёт"** и **"Контроль доставляемости"**

**Фильтры:**
- **Период** - диапазон дат "с" и "до", открывается календарь. Фильтрация по **дате вручения**

- **Клиент** (одиночный выбор) - выпадающий список клиентов с поиском

- **База** - переключение между Мск и Спб. Определяет, клиенты из какой базы отображаются в списке

- **Кнопка "Показать"** - отображает отчёт на основании выбранных фильтров

- **Кнопка "Скачать файлом"** - выгружает данные в `.xlsx`. Файл полностью соответствует таблице в веб-интерфейсе.

- **Кнопка "Контроль доставляемости"** - открывает страницу в новом окне со списком контрольных клиентов. Для каждого клиента можно задать пороговые значения выкупа. Если фактический процент по региону опускается ниже контрольного, в ячейке таблицы отображается иконка предупреждения.

На странице контроля доступны:
- Список клиентов с настроенными порогами
- Редактирование и удаление клиентов
- Кнопка **"Добавить"** - для добавления Клиентов

**Модальное окно добавления/редактирования:**

- Выбор клиента из выпадающего списка с поиском
- Таблица с пороговыми значениями:
1. Выкуп в заказах, %
2. Выкуп в деньгах, %

- Кнопка **"Сохранить"**

## Структура таблицы

Первая строка таблицы - **Общий показатель** по клиенту. Она всегда зафиксирована сверху и не смещается при сортировке остальных строк.


### Расчёты общего показателя (первая строка)
1. **Всего заказов** - сумма всех закрытых заказов по всем регионам, кроме статуса **Отмена**

2. **Выкуп в заказах, шт.** - сумма заказов со статусами **Доставлен**, **Частично доставлен** с датой вручения по всем регионам

3. **Выкуп в заказах, %** - (**Выкуп в заказах, шт.** / **Всего заказов**) x 100

4. **Выкуп в деньгах, руб.** - сумма итогового наложенного платежа по заказам **Доставлен** и **Частично доставлен** по всем регионам

5. **Выкуп в деньгах, %** - (Сумма первичного НП по всем закрытым заказам кроме статуса **Отмена** / **Выкуп в деньгах, руб.**) x 100

6. **Доставляемость** - общее количество заказов со статусами: **Доставлен**, **Частично доставлен**, **Оплата доставки**, **Отказ на месте**

### Строки по регионам
Каждая следующая строка относится к конкретному региону доставки, по которому зафиксирован факт доставки выбранного клиента.

1. **Регион** - регион доставки

2. **Всего заказов** - все закрытые заказы за период по региону, кроме статуса **Отмена**

3. **Выкуп в заказах, шт.** - количество заказов **Доставлен**, **Частично доставлен** с датой вручения по региону

4. **Выкуп в заказах, %** - (**Выкуп в заказах, шт.** / **Всего заказов**) x 100

5. **Выкуп в деньгах, руб.** - сумма итогового НП по заказам **Доставлен** и **Частично доставлен** по региону

6. **Выкуп в деньгах, %** - (Сумма первичного НП по всем закрытым заказам кроме статуса **Отмена** по региону / **Выкуп в деньгах, руб.**) x 100

7. **Доставляемость** - количество заказов по региону со статусами: **Доставлен**, **Частично доставлен**, **Оплата доставки**, **Отказ на месте**

8. **Скачать** - кнопка для экспорта заказов региона в `.xlsx`. В файл попадают заказы со статусами **Доставлен** и **Частично доставлен** за выбранный период.

**Cтолбцы таблицы:**
- Штрихкод
- Режим доставки
- Регион отправления
- Регион доставки
- Статус
- Наложенный платёж

## Пользовательский сценарий
1. Пользователь открывает раздел **Отчёты - Выкуп по Клиентам**

2. Выбирает клиента и период. (По умолчанию клиент не выбран - таблица отображается только после выбора клиента) 

3. Нажимает кнопку **"Показать"**

4. Формируется таблица, в которой указываются регионы, по которым осуществлялась доставка по данному клиенту за выбранный период

3. Пользователь может скачать отчёт:
   - **По региону** - кнопка **"Скачать"** в строке таблицы.
   - **Общий** - кнопка **Скачать отчёт** вверху страницы.

4. При необходимости в контроле доставляемости, пользователь нажимает кнопку **"Контроль доставляемости"**, добавляет/редактирует клиента и задаёт пороговые проценты. При нарушении порога в таблице появляется иконка предупреждения.

## Ограничения
- Один клиент на отчёт
- Поиск во всех выпадающих списках
- Не более одного уникального клиента в контроле
- Пороговые значения - целые числа от 0 до 100
- Период обязателен, по умолчанию - текущий месяц, конечная дата - не позднее вчерашнего дня
- Только заказы с датой вручения
- Режимы доставки:

| Мск база | Спб база |
| -------- | -------- |
| Обычная МСК и МО | Обычная МСК и МО |
| Экспресс МСК | Экспресс СПБ |
| Срочная МСК | Срочная SPB |
| ПВЗ МСК | ПВЗ МСК |
| Обычная СПБ и ЛО | Обычная СПБ и ЛО |
| ПВЗ СПБ | ПВЗ СПБ |
| Курьерская регионы | Курьерская регионы |
| ПВЗ Регионы | ПВЗ Регионы |
| Внутригородская Регионы | Внутригородская Регионы |
| Доставка МП | Доставка МП |
| Доставка КГТ | Доставка КГТ |
| Внутрирегиональная доставка | Внутрирегиональная доставка |
| Доставка КГТ Регион | Доставка КГТ Регион |

- Экспорт только в `.xlsx`
    - В файле - только статусы **Доставлен** и **Частично доставлен**

- **Иконка предупреждения** - только если фактическое значение меньше контрольного

## Не входит в разработку
- Комментарии к клиентам
- Массовая настройка выкупа по регионам
- Выбор/исключение регионов, режимов и статусов доставки в фильтрах
- Выбор нескольких клиентов
- Сравнение доставляемости
- Уведомления о низкой выкупаемости
- Графики с динамикой выкупа
