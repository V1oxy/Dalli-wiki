# Попадание в дату и интервал

## Цель
Разработка инструмента для анализа и контроля попаданий заказов в сроки доставки.

![](img/image_01.png)


# Правки текущего функционала
1. В столбце "Менеджер" заменить отображение КАМ-менеджера на отображение менеджера по сопровождению.

2. Объединить данные из баз Москва и Санкт-Петербург в единый набор данных для формирования отчёта.
   
3. Добавить возможность выбора периода по дате закрытия заказа.
   
4. Изменить источник получения даты закрытия заказа с поля "Дата вручения" на дату проставления финального статуса курьером в выдаче.

5. Изменить логику расчёта показателей по столбцам таблицы в соответствии с техническим заданием.
   
6. Добавить новые столбцы в соответствии с техническим заданием.


# Общее Техническое задание

## Основные задачи
1. Добавить новый раздел в Staff: **Отчёты - Попадание в дату и интервал**.
   
2. Создать панель управления с переключателями, фильтрами и кнопками действий.
   
3. Разработать таблицу с агрегированными данными.
   
4. Реализовать возможность экспорта данных в формат `.xlsx`.
   
5. Добавить возможность индивидуальной настройки по клиенту, для визуального выделения строк, если SLA опускается ниже заданного порога.
   
6. Добавить возможность оставлять комментарий по клиенту.


## Глоссарий

**Дата/Интервал доставки** - Дата/Интервал, указанные Интернет-Магазином при создании заказа.

**Дата/Время вручения** - Дата/Время указания курьером статусов:
- **Доставлено**
- **Частичный отказ**
- **Не доставлено** c причинами: "Отказ на месте.." ; "Оплата доставки"  

**SLA** - обязательства по соблюдению сроков доставки.

**Cтоимость выкупа** - сумма денежных средств, полученных от получателей.


## Исходные данные и ограничения

### Получение данных по дате и времени доставки
Дата и время закрытия заказа определяются по моменту проставления курьером финального статуса в выдаче.

### Объединение баз МСК и СПБ
1. Необходимо объединить данные по клиентам из баз МСК и СПБ в единую таблицу.
   
2. Для заказов, присутствующих одновременно в базах МСК и СПБ, отчёт должен объединить записи по штрихкоду, чтобы получить полную информацию о дате, времени и причине, указанной курьером.
   

---

- **РКО клиенеты ООО "ДС-СПБ":** 7%

## Функционал инструмента "Попадание в дату и интервал"

### Панель управления 
В верхней части страницы необходимо расположить **"Панель инструментов"**, позволяющую настраивать отображение данных в таблице. 

**Панель управления состоит из:**
- Переключателей
- Фильтров
- Кнопок действия

![](img/image_02.png)


**Переключатели**

- **По Клиентам / По Регионам** - определяет способ группировки данных в таблице.
  - **По Клиентам** - Вывод происходит в разрезе клиентов. Регионы агрегированы внутри карточки клиента.
  - **По Регионам** - Вывод происходит в разрезе регионов доставки. Клиенты агрегированы внутри региона.

- **По дате доставки / По дате вручения** - Выбор основания для расчёта **Периода**.
  - По дате доставки - Дата указанная Интернет-Магазином при создании заказа
  - По дате вручения - Дата указания курьером статусов: **Доставлено** ; **Частичный отказ** ; **Не доставлено** c причинами: "Отказ на месте.." ; "Оплата доставки"  
  
---

**Фильтры**

Все выбранные фильтры применяются одновременно. Если по заданным условиям заказы отсутствуют, таблица не формируется.

- **Период** - Диапазон дат "с" и "до". Выбор через всплывающий календарь.
  
- **Интервал** - Интервалов доставки. Задаётся двумя ячейками "с" и "до" с шагом 1 час (8:00–23:00). 
  - **"Все"** - отображаются заказы с любым интервалом доставки.
  - **При указании диапазона** - отображаются только заказы, интервал доставки которых, соответствует указанному диапазону.
  
- **Клиенты** -  множественный выбор (чипсы) с поиском внутри выпадающего списка. Поддерживается одиночное и массовое удаление. Пусто = все клиенты. При выборе - только указанные.
  
- **Исключённые клиенты** - множественный выбор (чипсы) с поиском внутри выпадающего списка. Пусто = никто не исключён. Не применяется, если заполнен фильтр **"По клиентам"**
  
- **Менеджеры** - множественный выбор (чипсы) с поиском внутри выпадающего списка. Пусто = все. При выборе - только клиенты указанных менеджеров по сопровождению.
  
- **Регионы** - множественный выбор (чипсы) с поиском внутри выпадающего списка. Пусто = все. При выборе - только указанные регионы доставки
  
- **Типы** - множественный выбор (чипсы, с поиском): A, B, C, D, VIP. Пусто = все. При выборе - только указанные типы клиентов.

---

**Кнопки**

- **Показать** - формирует таблицу в соответствии с заданными параметрами в панели управления.
  
- **Скачать файлом** - экспорт отчёта в `.xlsx` с содержанием тех же данных и структурой, что отображаются веб-таблице.
  
- **Контроль по SLA** - открывает страницу с настройками контроля SLA и добавления комментария по клиенту.


## Структура таблицы

### Правила таблицы
- При входе в раздел **"Попадание в дату и интервал"** таблица не формируется автоматически. Данные загружаются только при нажатии кнопки **"Показать"**.

- Данные в таблице формируются на основании настроек в **Панели управления**.

- Предусмотреть горизонтальную прокрутку таблицы для масштабирования без обрезки названий столбцов.
  
- **Первая строка таблицы:** **"Общий показатель"** - Отображает общую метрику по всем данным таблицы. Зафиксирована и не сортируется.

- **Последующие строки:**  Детализация, где каждая строка соответствует одному клиенту или одному региону, в зависимости от выбранного режима.

## Расчёт показателей таблицы

### Общие правила
- Все процентные показатели рассчитываются с точностью до двух знаков после запятой.
  
---

### СРЦ
`СРЦ = Cтоимость выкупа / Количество`

### СРЦ без НДС
`СРЦ без НДС = СРЦ / (1 + Ставка НДС)`


### Раньше даты, %
`Раньше даты / Количество × 100%`

### Попадание в дату, %
`Попадание в дату / Количество × 100%`

### Раньше интервала, %
`Раньше интервала / Попадание в дату × 100%`

### Попадание в интервал, %
`Попадание в интервал / Попадание в дату × 100%`


### Своевременно принятые заказы
`Количество − Заказы, полученные позже`

### Частота доставляемости в срок (дата)
`Количество / Позже даты`

Если `Позже даты = 0`, значение принимается равным **Количество**.

### Частота доставляемости в срок (дата), %
`Количество / Частота доставляемости в срок (дата) / Количество × 100%`

Если `Позже даты = 0`, значение принимается равным **100%**.

### Частота доставляемости в срок (интервал)
`Количество − (Позже даты / Позже интервала)`

Если `Позже интервала = 0`, значение принимается равным **Количество**.

### Частота доставляемости в срок (интервал), %
`Количество / Частота доставляемости в срок (интервал) / Количество × 100%`

Если `Позже интервала = 0`, значение принимается равным **0%**.



## Особенности при формировании "Общего показатели"
Количество заказов по всем столбцам суммируются. Процентные и расчётные показатели вычисляются заново на основании полученных сумм, а не усредняются по строкам.

Кнопка **"Скачать"** для строки "Общий показатель" не предусмотрена.

## Кнопка "Контроль по SLA"

### Назначение
Открывает раздел для индивидуальной настройки порогов SLA по клиентам. Если показатель опускается ниже заданного порога - строка подсвечивается.

### Раздел настройки

![](img/image_04.png)


Содержит таблицу с уже добавленными клиентами и столбцами:
1. Код клиента
2. Клиент
3. % Попадания в дату
4. % Попадания в интервал
5. Действия (Редактировать / Удалить)

Под таблицей - кнопка **"Добавить"**.

### Модальное окно (добавление / редактирование)

![](img/image_05.png)


- **Выбор клиента** (при редактировании подставляется автоматически значения выбранного клиента).
  
- **Пороговые значения:**
  - % Попадания в дату - ввод числа, знак `%` подставляется автоматически.
  - % Попадания в интервал - ввод числа, знак `%` подставляется автоматически.
- 
- **Комментарий** - текстовый редактор с возможностью форматирования текста

### Логика работы
Если в отчёте **"По Клиенту"** значение столбца **Попадание в дату, %** или **Попадание в интервал, %** ниже заданного порога:
- Строка окрашивается в красный.
- Появляется значок `!`, при наведении на который, отображается подсказка с требуемым значением.

![](img/image_06.png)


### Отображение комментария
Комментарий к клиенту показывается под таблицей, только если отчёт сформирован по одному клиенту (клиент выбран в фильтре).

## Пользовательский сценарий
1. Пользователь заходит в раздел **Отчёты - Попадание в дату и интервал**.
2. Выбирает режим отображения (По дате доставки / По дате вручения)
3. Выбирает режим отображения (По Клиентам / По Регионам).
4. Настраивает фильтры: Период, Интервал, Клиенты и т.д.
5. Нажимает **"Показать"** - формируется таблица.
6. При необходимости настраивает контроль SLA через кнопку **"Контроль по SLA"**.
7. Может скачать отчёт или детализацию по строке через кнопки **"Скачать файлом"** / **"Скачать"**.


## Не входит в разработку
- Выбор и исключение режимов доставки (Обычная МСК и МО, Курьерская Регионы и т.д.)
- Выбор определённых статусов курьера
- Детализация по партнёрским заказам
- Графики с динамикой доставляемости
- Уведомления в Telegram / Staff о низком SLA
